## 2. 修复用户头像显示问题

### 问题描述
除了"联系我们"界面外，其他用户界面（首页、商品列表、购物车、博客等）的左上角显示固定logo图片，而不是用户自己的头像。

### 根本原因
后端API返回的数据字段是 `data.data`（小写），但部分前端页面使用了 `data.Data`（大写D），导致无法正确获取用户信息。

### 修改文件
1. `src/main/resources/templates/webSite/index.html`
2. `src/main/resources/templates/webSite/shop.html`
3. `src/main/resources/templates/webSite/cart.html`
4. `src/main/resources/templates/webSite/blog.html`

### 具体修改
将所有页面的 `getUserInfo()` 方法中的 `data.Data` 统一修改为 `data.data`：

**修改前：**
```javascript
success: function (data) {
    if (data.code !== 500) {
        that.userInfo = data.Data  // 错误：大写D
    }
}
```

**修改后：**
```javascript
success: function (data) {
    if (data.code !== 500) {
        that.userInfo = data.data  // 正确：小写d
    }
}
```

### 影响范围
- 用户首页（/web/userindex）
- 宠物服务列表页（/web/userproductlist）
- 购物车页面（/web/usercart）
- 博客/公告页面（/web/userblog）

### 注意事项
- "联系我们"页面（contact.html）之前已经使用正确的 `data.data`，所以不需要修改
- 修复后，所有页面的左上角和右上角都会正确显示用户头像

## 技术细节

### 后端API数据格式
根据 `AjaxResult` 类的定义，后端返回的数据格式为：
```json
{
    "code": 200,
    "msg": "操作成功",
    "data": {
        // 实际数据内容
    }
}
```

因此，前端应该使用 `data.data` 来访问实际数据，而不是 `data.Data`。
